Lobby

A player is inserted into a random map, where it can move its units around the map. When he finds others there will be a possibility to join a game, by building or setting a flag. Each game should be left open so that late comers can also join.

Building units


A client can notify server that an unit has been built. Server will notify others in the game.
A client requests server to create a unit. Server notifies its clients about the new unit.

Animating frames

A loop of frames could be saved within one file.

Generic polling game networking protocol.

Each client polls periodically the server. On each poll any queued messages will be sent and any messages will be recieved. Once the polling stops, the server may destroy the session.

Methods which are availible for clients without session or with invalid sessions should be marked with an attribute SafeForAnonymousClients.

 

CheckVersion

If a browser has cached the client application, this will atleast stop the client with a message to clear the cache.

JoinLobby

Until this moment the client is considered to be anonymous. This will create its own session. A client with a session may time out and the session will be deleted. An anonymous client cannot receive any server sent messages.

DisplayText

Server sends text for the client to be displayed.

ClientReload

Forces the clients to reload. This may be used to indicate that the server has been set to work in slave mode, which can invalidate all given sessions.
